üéØCMT Format Overview

CMT encodes each musical event as a single token using this standardized format:

<Note><Accidental><Octave>:<DurationCode>

üí° Key Concept: Each token represents a complete musical event with both pitch information (using Scientific Pitch Notation) and timing information (using ABC notation-style duration codes).

üõëRepresenting Rests

Use "rest" as the pitch component to represent silence:

"rest:q" // quarter rest "rest:h." // dotted half rest "rest:w" // whole rest "rest:e" // eighth rest

üìù Note: Rest tokens follow the same duration encoding rules as pitched notes, but use the literal string "rest" instead of a pitch specification.

üß™Basic Examples

Here are some practical examples of CMT tokens in action:

[ "C4:q", // Quarter note C4 "F#3:e", // Eighth note F#3 "Bb5:h.", // Dotted half note Bb5 "rest:q", // Quarter rest "G4:tq" // Triplet quarter note G4 ]

‚úÖ Valid Examples

  • A4:q - Quarter note A4
  • C#5:h. - Dotted half C#5
  • rest:e - Eighth rest

‚ùå Invalid Examples

  • H4:q - Invalid note name
  • C4 - Missing duration
  • C:q - Missing octave

üß∞Optional Extensions

Velocity/Dynamics

Measure Markers

M1:C4:q M2:D4:h
